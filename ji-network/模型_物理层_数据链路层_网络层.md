  - 最基本功能：数据通信

07.A
- 通信子网
  - 对应OSI参考模型下三层
    - 物理层
    - 数据链路层
    - 网络层
  - 包含硬件设备：网桥、交换机、路由器
- 资源子网
  - 负责运行信息处理应用程序
  - 是网络中信息流动的源和宿
  - 提供可共享的硬件、软件和信息资源

08.A
- 以太网数据传输特点
  - 数据以帧形式传输
  - 源端长报文需分为若干数据块
  - 各层加上相应控制信息
    - 网络层：分组
    - 数据链路层：以太网帧
  - 用户在会话期间断续使用以太网链路

09.A
- 电路交换特点
  - 建立连接时延大
  - 传输期间占据链路
  - 实时性强，传输时延小
- 报文交换和分组交换特点
  - 基于存储转发交换
  - 无需事先建立连接
  - 动态分配通信资源
  - 线路利用率高
  - 存在存储转发时延
  - 有额外控制开销
  - 实时性较差

10.C
- 分组交换相对报文交换特点
  - 将报文划分为固定最大长度的分组
  - 以分组为单位进行传输

11.B
- 分组交换特点
  - 数据分成等长小数据段
  - 每段加上控制信息
  - 传送数据总开销大
  - 信道利用率高
  - 传播时延取决于传播介质和距离

12.A、A、A、C
- 数据交换方式特点比较
  - 电路交换
    - 优点：传输时延小、通信实时性强
    - 适用：交互式会话类通信
    - 缺点
      - 不适应突发性通信
      - 系统效率低
      - 无存储数据能力
      - 不能平滑网络通信量
      - 无差错控制能力
  - 报文交换和分组交换
    - 采用存储转发
    - 传输时延较大
    - 分组交换传输时延小于报文交换

13.A
- 数据报方式特点
  - 无连接的分组交换技术
  - 报文拆分为较小数据段
  - 加上地址等控制信息构成分组
  - 提供尽最大努力的交付
  - 不保证可靠性
  - 分组可能出错或丢失
  - 每个分组独立选择路由
  - 分组不一定按序到达

14.A
- 网络分类方式
  - 按分布范围：广域网、城域网、局域网、个人区域网
  - 按拓扑结构：星形网络、总线形网络、环形网络、网状网络

# 二、综合应用题  

01.【解答】
- 分组数据大小与开销分析
  - 计算条件
    - 分组数据大小为D
    - 分组数目=$10^{6}/D$
    - 开销=100×N
  - 开销计算
    - D=1000时
      - 总开销=$100×10^{6}/1000+1000=1010000\mathrm{B}$
    - D=20000时  
      - 总开销=$100×10^{6}/20000+20000=25000\mathrm{B}$
  - 最优值求解
    - 设总开销y=$10^{8}/D+D$
    - 求导得$\mathrm{d}y/\mathrm{d}D=1-10^{8}/D^{2}$
    - 当D=10^4时,$\mathrm{d}y/\mathrm{d}D=0$
    - 最佳分组大小为10000B

02.【解答】
- 传播时延计算
  - 基本参数
    - 距离=2×10^3m
    - 传播速度=2×10^8m/s
  - 传播时延=$10^{-5}\mathrm{s}=10{\upmu}\mathrm{s}$
- 带宽分析
  - 分组100B时
    - 设带宽为x
    - x=100B/10μs=10MB/s=80Mb/s
  - 分组512B时
    - 设带宽为y
    - y=512B/10μs=51.2MB/s=409.6Mb/s
  - 结论:带宽应分别为80Mb/s和409.6Mb/s

03.【解答】
- 网络传输策略分析
  - 易丢失分组情况
    - 采用逐一确认
    - 只重传丢失分组
  - 高度可靠情况
    - 文件结尾单次确认
    - 节省带宽
    - 单分组丢失需重传整个文件

# 2.1.5 答案与解析  

# 一、单项选择题  

01.D
- 信道与通信电路的区别
  - 一条双向通信电路包含两个信道
    - 发送信道
    - 接收信道
  - 多用户共用时每用户有一个信道
  - 调制是数据转模拟信号的过程
  - 波特率 = 比特率/每符号含比特数

02.A
- 信道最大传输速率影响因素
  - 信道带宽
  - 信噪比
    - 与平均信号功率和噪声功率比值有关

03.B
- 传输方式特点比较
  - 并行传输
    - 距离短
    - 速度快
  - 串行传输  
    - 距离长
    - 速度慢

04.B
- 曼彻斯特编码特点
  - 时钟和数据包含在信号中
  - 码元中间跳变作为时钟信号
  - 不同跳变方式作为数据信号
  - 每码元中间都有电平跳变
  - 最适合二进制数字信号传输

05.B
- 曼彻斯特编码优点
  - 用码元中间电平跳变表示比特
  - 方便收发双方同步时钟
  - 无需额外时钟信号

06.A
- 各种编码方式比较
  - 非归零编码
    - 最简单的编码方式
    - 低电平表示0,高电平表示1
    - 码元间无间隔标志
    - 不包含同步信息
  - 曼彻斯特编码和差分曼彻斯特编码
    - 码元分两等时间间隔
    - 中间跳变作为同步信息
    - 无需额外同步信息
    - 频带宽度是原始基带2倍

07.A
- 波特率定义
  - 表示信号每秒变化次数
  - 与比特率区别

08.B
- 以太网编码速率计算
  - 采用曼彻斯特编码
  - 每比特需两个电平表示
  - 波特率是数据率2倍
  - 数据率 = 40Mb/s/2 = 20Mb/s

> attention：
曼彻斯特编码的每个比特需要两个信号周期，信号率是数据率的2倍，编码效率是50%

09.D
- 码元与比特关系计算
  - 比特率 = 波特率 × $\log_2n$
  - k比特信息量需 $n=2^k$ 个离散值
  - 每码元比特数 = 4000/1000 = 4
  - 有效离散值个数 = $2^4 = 16$

  - 比特串可能是0011 0110或1100 1001

11.B
- 波特率计算
  - 码元取 $2^n$ 个离散值含n比特信息
  - 一码元含2比特信息
  - 波特率 = 比特率/每码元比特数
  - 波特率 = 64k/2 = 32kBaud

12.C
- 奈奎斯特定理应用
  - 最大数据传输速率 = $2W\log_2V$
  - 计算得48kb/s
  - 24kHz采样超过波特率上限16kBaud

13.D
- 信道最大传输速率计算
  - W = 4000Hz
  - 最大码元传输速率 = 2W = 8000Baud
  - 16种状态表示4比特数据
  - 最大传输速率 = 8000×4 = 32kb/s

14.B
- 信道容量限制计算
  - 香农定理限制:280000b/s
  - 奈奎斯特定理限制:8000b/s
  - 取两者较小值8000b/s

若给出码元与比特关系，需受两公式共同限制。详见本章疑难点。

18.B

  - 上述情况导致信道利用率低

- 停止-等待协议特点
  03.D
  - 发送方设置计时器
  - 发送帧后等待确认
  - 超时未收到确认则重传

- 停止-等待协议编号
  04.A

  - 相同序号数据帧表示超时重传
  - 相同序号确认帧表示收到重复帧

- 停止-等待协议计算
  05.D
  - 信道最大利用率计算
    - $U = \frac{L/C}{L/C + 2R} = \frac{L}{L + 2RC} = \frac{L}{L + 2 \times 30\text{ms} \times 4\text{kb/s}} = 80\%$
  - 得出帧长 $L = 960\text{bit}$

- 信道利用率计算
  06.B
  - 忽略确认帧发送时延
  - $\frac{\text{数据帧发送时延}}{\text{数据帧发送时延 + 往返时延}} = 0.4$
  - 数据帧发送时延为 $4/30\text{s}$
  - 数据帧长度为 $6\text{kb/s} \times 4/30\text{s} = 800\text{bit}$

# 3.5.5 答案与解析  

# 一、单项选择题  

01.A
- 介质访问控制特点分析
  - 随机访问特点：CSMA/CD协议
  - 集中式特点：主从式协议，主站控制从站发送顺序
  - 轮询访问特点：令牌传递协议

02.B
- 频分复用原理
  - 物理信道带宽大于单个信号带宽时
  - 将总带宽分割成若干子信道
  - 每个子信道传输一种信号

03.D
- TDM(时分复用)原理与条件
  - 基本原理
    - 发送端：不同用户信号交织在不同时间片
    - 接收端：提取各时间片信号还原
  - 必要条件
    - 介质位速率大于单个信号位速率
    - 介质带宽大于复合信号带宽

04.B
- TDM与FDM比较
  - TDM优势
    - 抗干扰能力强
    - 可遂级再生整形
    - 易实现自动转换
  - 适用场景
    - FDM适合模拟信号
    - TDM适合数字信号

05.B
- 时分复用类型
  - 同步时分复用：静态预分配时间片
  - 异步时分复用：动态分配时间片

06.A
- 信道分配方式比较
  - 静态划分：TDM，不发生冲突
  - 动态分配：ALOHA、CSMA、CSMA/CD，可能发生冲突
  - 轮询访问：不发生冲突

07.C
- 纯ALOHA协议特点
  - 站点可立即发送数据
  - 无需等待信道空闲
  - 无需等待时间槽
  - 无需发送RTS帧

08.C
- p-坚持CSMA协议特点
  - 检测到信道空闲后
    - 概率p发送数据
    - 概率1-p推迟到下一时隙
  - 优点
    - 降低冲突概率
    - 克服非坚持CSMA延迟长的缺点

09.C
- 非坚持CSMA工作流程
  - 发送前监听信道
  - 信道忙则放弃监听
  - 等待随机时间后再监听
  - 信道空闲则发送数据

10.B
解析同上。

11.A
- CSMA/CD争用期定义：信号在最远两端点间往返传输时间

12.B
- 冲突检测特点
  - 发送完帧前未检测冲突
  - 不能确定不会发生冲突
  - 原因：站点间往返传播时延可能大于帧发送时间

13.B
- CSMA/CD协议要求
  - 发送帧时间 ≥ 争用期时间
  - 高速传输应对方法
    - 增加最短帧长
    - 缩短电缆长度

14.D
- 最小帧长计算
  - 来回路程 = 10000×2m
  - 往返时间RTT = 10000×2/(200×10^6) = 10^-4s

- 二进制指数退避算法特点
  - 动态适应发送站点数量
  - 后退时延与重发次数关系
  - 负载影响

16.D

  - 随机整数选取
    - 范围：0至2^min(k,10)-1
    - k为冲突发生次数
  - 等待时间计算
    - 等待时间 = r×2t
    - t为以太网端到端延迟

17.B
- 100Mb/s以太网退避时间计算
  - 争用期：512b÷100Mb/s = 5.12μs

18.C

21.A

  - 站点间最长距离：200m

      - 允许多个移动站同时通信
      - 与有线局域网差异大

16.C
- 交换机转发行为分析
  - A→B：交换表空，全部泛洪
  - H→A：根据交换表定向转发
  - E→X：集线器全部转发
  - X→E：根据交换表处理

17.A
- 交换机PDU地址特性
  - 工作在数据链路层
  - 使用目的物理地址转发

18.B
- 直通交换延时计算
  - 只检查帧目的地址(6B)
  - 最短传输时延：$6{\times}8\mathrm{bit}{\div}100\mathrm{Mb/s}\!=\!0.48\upmu\mathrm{s}$

20.A
- 交换机本质特性

  - 可分割冲突域
  - 不能分割广播域

21.D

  - 集线器：不隔离冲突域，广播转发
